🎯 HOW TO ADD (Choose One Method)
────────────────────────────────────────────────────────────────────

METHOD A - XCODE UI:
  1. Select Reczipes2 target → Info tab
  2. Scroll to "Exported Type Identifiers" → Click +
  3. Fill in values from box #1 above
  4. Scroll to "Document Types" → Click +
  5. Fill in values from box #2 above

METHOD B - XML SOURCE:
  1. Right-click Info.plist → Open As → Source Code
  2. Copy XML from RecipeBook-Info.plist file
  3. Paste before closing </dict> tag
  4. Save

🧹 AFTER ADDING
────────────────────────────────────────────────────────────────────
  1. Clean Build Folder      (⇧⌘K)
  2. Delete app from device   
  3. Build and Run            (⌘R)

✅ VERIFY IT WORKS

🐛 TROUBLESHOOTING
────────────────────────────────────────────────────────────────────
  Issue: File won't open in app
    → Verify Info.plist entries match exactly
    → Clean build and delete app
    → Reinstall completely

  Issue: Still seeing ISSymbol errors
    → Delete app from simulator
    → Reset simulator (Device → Erase All)
    → Rebuild fresh

  Issue: Changes don't appear
    → Close and reopen Xcode
    → Try on real device instead of simulator

🎓 KEY CONCEPTS
────────────────────────────────────────────────────────────────────
  UTI = Uniform Type Identifier
    → System-level file type registration
    → Like file extensions but more powerful
    → Required for proper iOS file handling

  Export flow:
    App → .recipebook file → System recognizes → Share/Save

  Import flow:
    Tap file → System launches app → Import sheet → Done
